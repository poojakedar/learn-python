# Day 8 Concepts: Mini Projects

Day 8 is about combining all basics to build complete small apps.

## 1) What is a mini project

A mini project is a small real-world program.
It usually combines:

1. input
2. conditions
3. loops
4. functions
5. lists or dictionaries
6. file handling

## 2) Project structure idea

Most beginner projects can use this structure:

1. helper functions
2. main() function
3. loop/menu to interact with user
4. file save/read at the end

## 3) Why main function is useful

Using main() keeps program organized.
You can read your code from top to bottom more clearly.

## 4) Menu-driven program

Common beginner pattern:

1. Show menu options
2. Take user choice
3. Run matching block with if/elif
4. Repeat until user chooses Exit

## 5) Data design choices

Use list when:
- You need ordered items

Use dictionary when:
- Each item has named fields
- Example: {"amount": 120, "category": "food"}

## 6) Validation and error handling

Always validate user input in projects.
Use try/except to prevent crashes on wrong input.

## 7) Save project output

Writing to a file makes app practical.
Examples:
- expense report
- student marks
- to-do list

## 8) Common beginner mistakes

1. Putting all code in one huge block
2. Not using functions
3. No input validation
4. Forgetting to test edge cases (empty list, wrong choice)

## 9) Day 8 goal

By end of Day 8, you should be able to build one app with:

1. at least 3 functions
2. one loop
3. condition logic
4. list or dictionary data
5. save output to text file
